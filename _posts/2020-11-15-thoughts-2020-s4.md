对产品缺陷的敏感会随时间不断减弱。

最该学的语言或许是 Java C++ 与 JS，然后 Python 万能辅助。

对于寻找适合的工具，应该先动手做事情，把遇到的问题准确的描述成需求，然后按照需求去寻找合适的工具。这对于学习编程语言也是类似的，可以先找文档或者书本浏览目录结构，然后动手做事情，动手的过程中发现问题，定义问题，然后寻找合适的工具或者创建需要的工具。

想要传播有价值的信息，哪里权重高，就去哪里发出信息。

点号：句号（ 。）、问号（ ？）、叹号（ ！）、逗号（ ，）顿号（、）、分号（；）和冒号（：）。

标号：引号（“ ” ‘ ’）、括号〔（ ） [ ] { } 〕、破折号（ —— ）、省略号（······）、着重号（ ．）、书名号（《 》〈 〉）、间隔号（·）、连接号（ — ）和专名号（ ____ ）、分隔号（/）。

一些开源项目的一个典型特征是支持自定义配置，从而可以适应更多情况，产生更多价值。但商业项目未必需要这样做，价值反而来自开箱即用与强大的售后服务。

笔记和写作不同，笔记要尽可能多的突出链接的作用，Bear 和各类笔记的`[[` 要多用起来。

对用户需求高度敏感警觉，是未来一段时间需要时刻注意的，价值会从里面产生。

原子组合不应该单纯是流程上打散到原子来组成整体，更时不时考虑复用性与抽象，否则对于设计与开发效率没有助益。

发现了一种快速确定“新的库用什么版本？”的解决方法，去找一个使用了这个库的热门仓库，然后看项目构建文档里的版本号。
